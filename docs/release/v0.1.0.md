# v0.1.0 – Initial release

## Features

- WeChat Mini Program document cropper component
- Single-page workflow: choose image → adjust crop → export
- Multi-page workflow: add pages, switch, delete, export all
- Manual corner / edge / area adjustment
- Experimental post-capture boundary suggestion (falls back to default 10% inset on low confidence)
- Local Canvas crop export (JPEG, max 3000px side)
- No server, OCR, or cloud dependency — fully offline
- 7 component events: ready, detectstart, detectcomplete, detectfallback, change, crop, error
- Public methods: getPoints(), exportCrop()
- Coordinate mapper: aspect-fit display ↔ original image pixel mapping
- Document boundary detector with 12-scenario test coverage
- Security scan with sensitive data pattern detection
- Cross-platform CI runner (node scripts/run-ci.js)

## Known limitations

- Automatic boundary detection is **experimental** — works best on high-contrast images
- No perspective correction — rectangular crop only
- Complex backgrounds usually require manual adjustment
- Real-device compatibility still needs broader community testing
- Auto-detect currently falls back to default inset in most synthetic test scenarios

## Tested

- 5 Node.js test suites, 42 assertions pass
- WeChat DevTools compile: pass
- Single page: choose → drag corner → drag edge → pan → export — verified
- Multi page: add 3 → switch → delete → export all — verified
- Console: no new red errors

## Install

```bash
npm install wechat-miniapp-document-scanner
```

Or copy `miniprogram-component/document-cropper/` into your WeChat Mini Program project.

## Demo

Import `examples/wechat-miniprogram-demo/` into WeChat DevTools, replace `appid` in `project.config.json` with your own.
